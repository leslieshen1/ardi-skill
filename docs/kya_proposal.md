# KYA：Agent 准入门槛方案

## 背景

当前子网（Predict、Mine 等）没有注册成本和信用机制，大量脚本批量注册 agent 抢占本属于真实用户的奖励。我们需要一套准入机制来解决这个问题。

---

## 方案概述

方案分两层：

- **协议层**：链上质押门槛，由合约强制执行，任何人无法绕过
- **服务层**：我们作为服务商提供代质押便利，降低真实用户的准入成本

两层配合实现"安全性不打折，用户体验不降级"。

---

## 一、协议层：质押准入

### 规则

| 项目 | 说明 |
|------|------|
| 质押金额 | 10,000 AWP |
| 触发时机 | Agent 注册时 |
| 锁定期 | 用户自选（如 30 天、90 天） |
| 工作期限 | 锁定期 = 工作期，到期必须续费 |
| 退出 | 锁定期满后可取回全部质押 |
| 惩罚 | 无 slash，纯门槛 |
| 与 veAWP 关系 | 可复用现有 veAWP 持仓 |
| 授权代理 | 支持第三方为指定 agent 地址质押 |

### 为什么有效

- 批量注册 100 个 agent = 锁定 100 万 AWP，经济成本直接拦住大部分脚本
- 时间锁防止资金轮转（不能质押→注册→解锁→再注册下一个）
- 持续续费带来长期成本压力

### 合约实现方向

- 改造 AWPRegistry 或新建独立质押合约
- 支持 `stakeFor(agentAddress, duration)` —— 第三方代质押授权
- 复用 veAWP：如果 agent 绑定的 veAWP 持仓满足金额和剩余锁定期要求，视为已质押
- Agent 工作资格查询接口，供子网合约/后端调用

---

## 二、服务层：代质押 + 身份验证

### 核心流程

```
用户选择验证方式 → 完成验证 → 我们代质押 10K AWP → 授权给用户 agent → agent 激活
```

### 身份验证方式（可插拔）

| 方式 | 说明 | 强度 |
|------|------|------|
| 发推文 | 用指定文案发推，我们验证 | 轻量 |
| KYC | 身份证/护照验证 | 强 |
| 其他 | 未来可扩展（邮箱、GitHub 等） | - |

验证的本质是证明"这个 agent 背后是一个真实的人"。

### 经济模型

- 我们的成本：质押资金的流动性时间占用
- 我们的收入：向用户收取利息
- 风险：极低，资金锁在协议合约中而非转给用户

### 用户视角

对于不想自己买币质押的用户：
1. 注册 agent
2. 选择验证方式并完成（如发一条推文）
3. 支付利息
4. 开始工作赚奖励

对于愿意自己质押的用户：
1. 持有/购买 AWP
2. 自行质押（或复用 veAWP）
3. 直接开始工作

---

## 三、未来方向：信用机制

在准入门槛之上，未来可叠加信用体系：

- 基于链上行为的信用分（活跃时长、任务完成率等）
- 信用高的 agent 可享受更低质押要求或更高奖励权重
- 信用低的 agent 被限制或踢出

此部分暂不设计，先把准入门槛跑通。

## 待讨论问题

### 合约层

1. **质押金额**：10,000 AWP 是否合适？太高影响真实用户，太低拦不住脚本
2. **最短锁定期**：是否需要设置下限（如最少 7 天）？防止极短锁定降低攻击成本
3. **复用 veAWP 的判定逻辑**：剩余锁定期不足时怎么处理？是否要求 veAWP 剩余期 ≥ 所选工作期？
4. **合约架构**：改造 AWPRegistry 还是独立合约？独立合约耦合低但多一次交互
5. **到期处理**：锁定期满后 agent 是自动失效，还是有宽限期？
6. **子网如何查询**：子网合约直接调用，还是通过 indexer/API 查？

### 服务层

7. **利息方式**：预付（注册时一次性付清）？从奖励中扣？按周期收？
8. **利息定价**：固定费率还是动态？参考什么基准？
9. **推文验证方案**：用 Twitter API（有成本、有限制）还是爬虫/第三方服务？
10. **推文被删**：用户验证后删推怎么办？是否需要定期复查？
11. **代质押上限**：我们最多同时为多少 agent 代质押？资金池如何管理？
12. **违规处理**：如果代质押的 agent 被发现是脚本，我们是否有权提前撤销授权？合约是否支持？

### 产品层

13. **多链支持**：质押在哪条链上？是否所有链统一？
14. **用户沟通**：如何向现有用户解释这个变化？是否有过渡期？
15. **优先级**：先在哪个子网试点？Predict 还是 Mine？
